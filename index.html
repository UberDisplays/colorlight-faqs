Quick Boot-Up Sequence (Always follow this order)

Signal source (PC/player) → Video processor → Sending card → Receiving cards / LED wall. Always power down in reverse order. Wrong sequencing is a top cause of no-signal issues.

🔧 Troubleshooting Common Problems

⬛ Black Screen / No Output

Fixes to try in order:
  • Follow correct power-on sequence: signal source → processor → sending card → wall
  • Check receiver card indicator lights (see LED Indicators section below)
  • Verify Ethernet cables between sending card and receiving cards are seated and undamaged (use Cat5e/Cat6)
  • Check brightness isn't set to 0 and screen isn't in blackout/test mode in LEDSetting
  • Confirm signal source resolution and refresh rate match processor input settings
  • Reload the .RCVBP configuration file on the receiving cards via LEDSetting
  • Update firmware on sending and receiving cards to latest version

📶 No Signal / Signal Loss

Fixes to try in order:
  • Check HDMI/DVI/SDI cable at processor input — swap cable to rule out failure
  • Verify processor input port matches selected input source in software
  • Confirm signal source is outputting at a supported resolution (1920×1080 @ 60Hz is safest)
  • Check if sending card driver software has crashed — restart LEDSetting/iSet
  • Verify the sending card is listed/detected in the software's device list
  • Try a different port on the sending card if multiple are available

🌊 Flickering / Unstable Image

Fixes to try in order:
  • Check all Ethernet cable connections — wiggle cables to test for intermittent contact
  • Confirm power supply voltage is stable and not under-loaded (use a multimeter)
  • Reduce refresh rate in LEDSetting if wall hardware doesn't support the current rate
  • Check for damaged or corroded RJ45 connector pins
  • Verify scan mode and OE polarity match the LED cabinet's hardware requirements
  • Replace Ethernet cable between sending and receiving cards with Cat6 shielded

🎨 Color Issues / Wrong Colors

Fixes to try in order:
  • Check RGB data polarity settings in LEDSetting — may need to be inverted
  • Verify color depth setting matches the LED driver IC specifications
  • Run ColorAdept calibration to correct color balance
  • Check if a color temperature preset was accidentally applied in the processor
  • Confirm the correct cabinet configuration file (.RCVBP) is loaded for your exact panel model

🔲 Partial Screen / Dead Sections

Fixes to try in order:
  • Check Ethernet daisy-chain — if one cable is bad, all cards downstream lose signal
  • Test each receiving card's indicator light independently to isolate the failure point
  • Verify the total pixel load per network port doesn't exceed the sending card's capacity
  • Check if dead section corresponds to a specific Ethernet port on the sending card
  • Re-seat or replace the receiving card in the dead cabinet

💻 Software Can't Find Device / Not Connecting

Fixes to try in order:
  • Ensure PC network adapter is set to same subnet as the sending card (default: 192.168.0.x)
  • Disable Windows Firewall temporarily to test connectivity
  • Check sending card driver is installed and recognized in Device Manager
  • Use a direct Ethernet connection (not through a managed switch) for initial setup
  • Restart LEDSetting/iSet after reconnecting hardware
  • Check USB connection if using a USB sending card (not Ethernet-based)

🌫️ Flowerscreen / Garbled Image (花屏)

Fixes to try in order:
  • This is almost always a configuration file mismatch — load the correct .RCVBP file for your panel
  • Check scan mode setting (1/4, 1/8, 1/16, 1/32 scan) matches cabinet hardware
  • Verify data polarity and OE polarity settings match the driver IC datasheet
  • Confirm cabinet width × height pixel dimensions are correctly entered in LEDSetting
  • Use LEDSetting's "Intelligent Setting" feature to auto-detect parameters

🚦 Receiving Card LED Indicator Guide

Colorlight receiving cards have red and green indicator lights. Learn to read them for instant diagnosis.

Red Light Green Light Meaning Action
OFF Any No power to receiving card Check power supply voltage (typically 5V)
ON (solid) OFF Power OK — No data signal from sending card Check Ethernet cable and sending card output
ON Very slow flash (1×/5 sec) Power OK — No network communication Check sending card and Ethernet cable in the daisy chain
ON 3 quick flashes then pause No video source input to sending card Check signal source connection and input selection on processor
ON Normal fast flash Normal operation — receiving data System healthy
ON Solid ON Receiving card in test pattern mode Exit test mode in LEDSetting software

🖥️ Programming & Configuration Walkthrough

Step 1: Hardware Connections

Connect signal source → processor input. Connect processor output → sending card input. Run Cat5e/Cat6 from sending card ports to receiving card input. Daisy-chain receiving cards across cabinets.

Step 2: PC Network Setup

Set PC NIC to static IP: 192.168.0.100, subnet 255.255.255.0. Colorlight sending cards default to 192.168.0.x range. Disable other network adapters to avoid conflicts.

Step 3: Install LEDSetting

Download and install LEDSetting from the official site. Run as Administrator. Open the software and confirm the sending card is detected under "Hardware" tab.

LEDSetting — Cabinet Configuration Process

  1. Open LEDSetting → click Hardware Manager → confirm sending card detected
  2. Click Receiving Card → select your receiving card model (e.g., i5A, K8, E320)
  3. Click Load from File and select the correct .RCVBP configuration file for your LED cabinet model
  4. If no file available, use Intelligent Setting: enter cabinet pixel width × height, select scan mode and driver IC type from the cabinet spec sheet
  5. Click Send to Hardware — the screen should display correctly. Check for flowerscreen (if so, review scan mode)
  6. Go to Screen Connection tab — define how cabinets are physically laid out (rows × columns, connection direction)
  7. Set input resolution and map it to the screen area in Sender Config
  8. Save the configuration and write to flash/EEPROM so it persists through power cycles

iSet — Quick Setup for Video Processors

  1. Connect PC to processor via Ethernet — open iSet software
  2. Select the processor model from the device list
  3. Set Input: choose source (HDMI/DVI/SDI), confirm resolution and refresh rate
  4. Set Output: define the output canvas size matching your LED wall total pixel dimensions
  5. Configure Scaling: choose fill mode (stretch, letterbox, native) to match your content
  6. Adjust Brightness, contrast, and color settings as needed
  7. Save preset — name it clearly (e.g., "Stage Left 1920x1080 HDMI")
⚠️
Configuration File Tip: Always keep a backup copy of your .RCVBP files and processor presets on a USB drive on-site. A corrupted or missing config file is the #1 cause of emergency troubleshooting calls.

A200 — Full Setup Walkthrough

  1. Connect A200 power. Connect A200 Ethernet output ports to receiving card inputs using Cat5e/Cat6 cables.
  2. Connect A200 to your network via LAN cable (or configure WiFi later). Connect PC to same network.
  3. Download and install PlayerMaster on your Windows PC from en.colorlightinside.com
  4. Open PlayerMaster → the A200 should appear automatically if on the same LAN. If not, click Search Device.
  5. Click the A200 in the device list → go to Screen Settings → load your .RCVBP cabinet config file (same as you'd use in LEDSetting)
  6. Set screen width × height in pixels, connection topology (cabinet layout), and brightness
  7. Go to Program tab → click Add Program → add your media files (video, images, text)
  8. Set program duration and layout zones if using split-screen
  9. Click Publish → content is sent to the A200 and plays immediately
  10. To schedule content: go to Schedule tab → set time-based rules for which programs play when
🔑
Default Advanced Settings Password: 168 — used in PlayerMaster to access screen parameter settings and advanced config options.

A200 Troubleshooting

🔍 A200 Not Found in PlayerMaster

Fixes to try in order:
  • Confirm PC and A200 are on the same LAN subnet
  • Disable Windows Firewall temporarily and re-search
  • Try connecting A200 directly to PC via Ethernet (no switch)
  • Check A200 indicator light — solid green = powered and booted correctly
  • Try connecting via USB cable instead of LAN for initial setup
  • Restart the A200 and wait 60 seconds before searching again

📡 A200 Online but Content Not Playing

Fixes to try in order:
  • Confirm the program was successfully Published (check publish status in PlayerMaster)
  • Check that screen parameters (resolution, cabinet config) are correctly saved on the device
  • Verify media file formats are supported: H.264/H.265 MP4, JPG, PNG, BMP
  • Check schedule — if a schedule is active, it may be overriding manual playback
  • Check storage — A200 has 8GB internal; delete old programs if storage is full

🌐 A200 Not Connecting to ColorlightCloud

Fixes to try in order:
  • Confirm A200 has internet access (ping test from the device network)
  • In PlayerMaster → Network → verify correct cloud server address is entered
  • Check account credentials for ColorlightCloud / LEDMASTER platform
  • Verify the device is registered/added to your cloud account
  • Try switching from WiFi to wired LAN for more stable cloud connectivity

🌐 ColorlightCloud & LEDMASTER Platform

Colorlight offers two cloud management platforms for remotely controlling and scheduling content on multiple displays from anywhere.

ColorlightCloud

The main cloud SaaS platform. Manage unlimited screens remotely — push content, set schedules, monitor device status, and get alerts via email. Role-based user management for teams.

Access: ColorlightCloud Product Page

LEDMASTER

Colorlight's cloud server solution for larger deployments. Centralized device management, content distribution, and monitoring. Can be self-hosted or cloud-hosted.

Access: ledmaster.colorlightcloud.com

PlayerMaster (LAN Mode)

Use PlayerMaster locally without cloud — manage all A-series players on your LAN directly. No internet required. Switch between LAN and Cloud mode in the software settings.

Best for: On-site management, venues without reliable internet

ColorlightCloud — Getting Started

  1. Register at the ColorlightCloud platform — contact your Colorlight distributor for account setup or register at the portal
  2. In PlayerMaster on your PC: click the Cloud button in the bottom toolbar → enter your account, password, and server address → click Login
  3. Add your A-series devices to the cloud account — devices must have internet access (LAN or WiFi)
  4. Go to Device tab — your screens appear here. Organize them into Groups (e.g., by location, zone, client)
  5. Go to Program tab → create programs with your media content (video, images, text tickers)
  6. Go to Schedule tab → assign programs to screens or groups on a time-based schedule
  7. Click Publish — content pushes to all assigned screens over the internet
  8. Monitor device online/offline status, storage, and playback from the Device dashboard

⚙️ Software Reference

LEDSetting

Primary tool for configuring receiving cards and LED cabinets. Use this to load .RCVBP files, set scan parameters, map screen layout, and save configs to hardware.

Replaces: LEDVISION (discontinued)

iSet

Video processor control software. Set input sources, output resolution, brightness, scaling, and presets. Use for real-time monitoring and adjustments during events.

Works with: Z-series, VX-series processors

PlayerMaster

Windows-based cloud player management software. Manage media, programs, schedules, and screen parameters for Colorlight cloud players (A-series, C3 Pro).

Use for: Digital signage, scheduled playback

ColorAdept

Professional calibration software for color accuracy. Corrects uniformity, white balance, and brightness across the LED wall using measurement hardware.

Requires: Colorlight calibration camera

GrandShow

Content creation and playback software for media servers (CS16K, CS20-8K Pro). Manage layers, effects, and live inputs for complex event productions.

Use for: Live events, media server control

LED Assistant (Mobile)

Mobile app (iOS/Android) for controlling Colorlight cloud players and basic screen adjustments from your phone on the go.

Download: App Store / Google Play
