A200 — Hardware Setup

  1. Connect A200 to power (12V DC). Wait ~60 seconds for full boot.
  2. Connect A200 Ethernet output ports to receiving card inputs using Cat5e/Cat6. The A200 has 4 output ports — connect them to your cabinets in order, daisy-chaining across rows.
  3. For initial configuration: connect A200 to PC via USB cable (comes in box). This is required to run PlayerMaster for first-time setup.
  4. For screen control cable (not content): if your wall uses a separate sending card alongside the A200, connect that separately. The A200 itself IS the sending card.
  5. Connect A200 to your network via Ethernet or configure WiFi through PlayerMaster (see Network Setup section).

A200 — Screen Parameter Setup in PlayerMaster

⚠️
You must set screen parameters on the A200 before content will display correctly. Do this via USB connection first.
  1. Connect A200 to PC via USB → open PlayerMaster → device appears under USB tab
  2. Click the device → go to Screen Settings tab on the right panel
  3. Click Advanced Settings → enter password 168
  4. Load the .RCVBP cabinet configuration file for your specific LED panel model (same file used in LEDSetting)
  5. Set Screen Width × Height in pixels — total pixels of the full wall, not a single cabinet
  6. Set cabinet layout: define rows and columns, and the cable connection direction (e.g., left to right, snake pattern)
  7. Set Brightness (0–100) and Refresh Rate to match your panel spec
  8. Click Send to Hardware — wall should display correctly. If garbled (flowerscreen), check scan mode in the .RCVBP file
  9. Click Save to Flash so settings persist through power cycles

A200 — Publishing Content via PlayerMaster

  1. In PlayerMaster → go to Program tab → click New Program
  2. Set program width × height to match your screen pixel dimensions
  3. Add a Page → add a File Window → drag in your media (MP4 H.264/H.265, JPG, PNG, BMP)
  4. Set play duration for each asset. For multiple assets, set play order
  5. Click Save → then click Publish → select your device → confirm
  6. Content transfers to A200 internal storage (8GB). Playback begins immediately after transfer
  7. To schedule: go to Schedule tab → set day/time rules for which program plays when
💡
Supported formats: Video — H.264 or H.265 MP4 (4K capable). Images — JPG, PNG, BMP. Text — built-in text ticker tool. Do not use H.263, AVI, WMV or MOV — these will not play.

📡 A200 Network Connection — All 4 Methods

You must configure network to use LAN or Cloud mode. For direct USB-only control, skip this. Always connect via USB first to configure network settings through PlayerMaster.

⚠️
First step for all methods: Connect A200 to PC via USB → open PlayerMaster → select device under USB tab → click the Network icon at the bottom toolbar → choose your method below.

📶 Way 1 — Player connects to WiFi

LAN ✓Cloud ✓
  1. In Network settings → select WiFi
  2. Choose your network name from the list
  3. Enter WiFi password → click Apply
  4. Click Test Network State
  5. Wait for "Connection successful"
  6. Device now shows under LAN or Cloud tab

📱 Way 2 — PC connects to A200 Hotspot

LAN ✓Cloud ✗
  1. A200 hotspot is ON by default
  2. Default SSID: player model (e.g., A200-XXXX)
  3. Default password: 123456789
  4. Connect your PC WiFi to the A200 hotspot
  5. Open PlayerMaster → device appears under LAN tab
  6. Good for on-site work with no router available

🔌 Way 3 — Both on same router/switch

LAN ✓Cloud ✓
  1. Plug A200 into router/switch via Ethernet
  2. Plug PC into same router/switch
  3. In Network settings → click Connect
  4. Device appears under LAN tab
  5. Most reliable — best for permanent installs
  6. Internet via router = Cloud mode also available

🔗 Way 4 — Direct PC to A200 via Ethernet

LAN ✓Cloud ✗
  1. Ethernet cable directly from PC NIC to A200
  2. In Network settings → select LAN
  3. Enter A200's IP address manually
  4. Click Connect → wait for confirmation
  5. Device appears under LAN tab
  6. No internet = Cloud mode not available
✅
Confirmed connected when: Device appears under the LAN tab (for Ways 2, 3, 4) or Cloud tab (Way 1 or 3 with internet) in PlayerMaster, and device info loads in the right panel.

🌐 LEDMaster Cloud — Full Setup & Management

🔑

What you need before starting

A LEDMaster/ColorlightCloud account — registration is NOT self-service. Contact your Colorlight distributor or email colorlight@lednets.com. You'll receive a Username, Password, and Server URL. The A200 must have internet access before it can connect to Cloud mode.

Part 1 — Set Up Device in Cloud Portal (Browser)

  1. Open browser → go to ledmaster.colorlightcloud.com → log in
  2. Left menu → Terminal Management → click New
  3. Fill in terminal name (e.g., "Lobby Screen 1"), select a Terminal Group
  4. Write down the Terminal Account — this is the LED player's login credential for the cloud server
  5. Click Submit — new terminal appears gray/offline in the list
  6. To group screens: Terminal Group → New Terminal Group — organise by location, floor, or client

Part 2 — Link A200 to Cloud via PlayerMaster

  1. Ensure A200 is networked with internet access (Way 1 or 3 from Network section)
  2. Connect A200 via USB → open PlayerMaster
  3. Click the Cloud button/icon in the PlayerMaster toolbar
  4. Enter your Account, Password, and Server URL → click Login
  5. Your terminal groups appear in the left panel under Cloud tab
  6. Find the A200 under the USB tab → drag it into the correct Cloud group
  7. Device registers to cloud — if info panel loads on the right, connection is confirmed
  8. Device now shows green/online in the browser portal

Part 3 — Upload Media & Publish Programs via Browser

  1. In LEDMaster portal → Media Management → File Upload → select your media files (max 512MB per file)
  2. Program Management → New Program → set width × height to your screen resolution
  3. Click Edit Contents → add a Page → add Windows (File Window, Text Window) → add media assets to each window
  4. Set play duration per asset → adjust play order if needed → click the upload/save icon
  5. Back in Program Management → select your program → click Publish
  6. Choose to publish to a Terminal Group (all screens) or a specific terminal
  7. Monitor download progress in the publish window — content downloads to device over internet
  8. To schedule: Terminal Management → Schedule → select terminal group → set time, day, and program

Remote Controls Available in Cloud Portal

📸 Screenshot

See exactly what's on the screen right now — without being on-site. Green border = device online. Click terminal name → Screenshot.

🔄 Reboot Device

Remotely restart the A200. Use when content is frozen or device is unresponsive but still showing as online in the portal.

🔀 Switch Program

Instantly swap the active program on any screen. Click device → Device Content Management → switch/delete programs.

☀️ Brightness / Volume

Adjust brightness, color temperature, and volume remotely. Useful for day/night automation without a schedule.

⬆️ Firmware Upgrade

Push firmware updates to devices remotely. Click device → Upgrade Terminal → select the correct firmware package.

📋 Device Logs

View device status, online/offline history, playback logs, and network info. Essential for remote diagnosis of problems.

🔧 Troubleshooting — Common Problems

⬛ Black Screen / No Output

Check in this order:
  • Check receiver card indicator lights — red light OFF means no power to card (check PSU voltage)
  • Verify Ethernet cables between sending card and receiving cards are firmly seated (Cat5e/Cat6)
  • Check brightness isn't set to 0% in LEDSetting or iSet — easy to accidentally zero out
  • Confirm the screen isn't in blackout/test mode in software
  • Reload the .RCVBP configuration file to the receiving cards via LEDSetting
  • Try a different Ethernet output port on the sending card
  • Update firmware on sending and receiving cards to the latest version

📶 No Signal / Signal Loss

Check in this order:
  • Swap the HDMI/DVI/SDI cable at the processor input — cable failure is the most common cause
  • Verify processor input selection matches your source (check iSet or front panel button)
  • Confirm source resolution is supported: 1920×1080 @ 60Hz is the safest default
  • Check if sending card driver/software has crashed — restart LEDSetting or iSet
  • Verify sending card appears in the software's device list — if not, check USB/Ethernet connection from PC
  • Try a different input port on the processor if available

🌊 Flickering / Unstable Image

Check in this order:
  • Wiggle all Ethernet cables while watching the screen — intermittent contact will show as flickering
  • Check power supply voltage with a multimeter — under-voltage causes flicker (typical: 5V or 3.8V per cabinet)
  • Reduce refresh rate in LEDSetting if the panel hardware doesn't support the current rate
  • Check for corroded or bent RJ45 pins — replace any suspect connectors
  • Verify scan mode and OE polarity match the LED cabinet hardware spec
  • Replace Ethernet cable between sending and receiving cards with Cat6 shielded cable
  • Check ground connections — floating ground can cause high-frequency flicker

🌫️ Flowerscreen / Garbled Image (花屏)

Check in this order:
  • This is almost always a config file mismatch — load the correct .RCVBP file for your exact panel model
  • Verify scan mode (1/4, 1/8, 1/16, 1/32) matches the cabinet's driver IC and PCB layout
  • Check data polarity and OE polarity — try inverting one or both if garbled in a specific way
  • Confirm cabinet pixel width × height are correctly entered — a single pixel off causes garbling
  • Use LEDSetting's Intelligent Setting to auto-detect parameters if no spec sheet available
  • Check the driver IC type is correctly selected for your specific panel

🎨 Color Issues / Wrong Colors

Check in this order:
  • Check RGB channel order in LEDSetting — may need to swap R and B channels
  • Verify color depth setting matches the LED driver IC specifications (typically 8-bit or 16-bit)
  • Run ColorAdept calibration to correct color balance across the wall
  • Check if a color temperature preset was accidentally applied in iSet or the processor
  • Confirm the correct .RCVBP file is loaded — wrong file can cause wrong color mapping
  • Check if individual modules have dead or dim LEDs — may require module replacement

🔲 Partial Screen / Dead Section

Check in this order:
  • Check Ethernet daisy-chain — one bad cable kills all cards downstream from that point
  • Watch receiver card indicator lights along the chain to find exactly where signal drops out
  • Check total pixel load per port — don't exceed the sending card's per-port pixel capacity
  • Confirm the dead section maps to a specific Ethernet output port on the sending card
  • Re-seat the receiving card in the dead cabinet — cards can loosen from vibration
  • Swap in a known-good receiving card to isolate hardware vs. config failure

💻 Software Can't Find Device / Not Connecting

Check in this order:
  • Set PC network adapter to static IP on same subnet as sending card (default: 192.168.0.x / 255.255.255.0)
  • Disable Windows Firewall temporarily to test — firewall blocks are a very common cause
  • Confirm sending card driver is installed and showing in Device Manager (no yellow warning icons)
  • Use a direct Ethernet connection (no managed switch) for initial setup
  • Restart LEDSetting/iSet after reconnecting hardware
  • Check USB cable if using USB-based sending card — try a different USB port or cable

📡 A200 Not Found in PlayerMaster

Check in this order:
  • Confirm PC and A200 are on the same LAN subnet (or use USB cable for initial access)
  • Disable Windows Firewall temporarily and click Search Device again
  • Try connecting A200 directly to PC via USB cable — this bypasses all network issues
  • Check A200 status LED — should be solid green when booted correctly
  • Restart the A200 and wait 60 seconds before searching again in PlayerMaster
  • Try connecting via the A200's built-in WiFi hotspot (SSID: device model, password: 123456789)

📡 A200 Online but Content Not Displaying

Check in this order:
  • Confirm program was successfully Published — check publish status/progress in PlayerMaster
  • Verify screen parameters (resolution, .RCVBP config) are saved and correct on the device
  • Check media file format — only H.264/H.265 MP4, JPG, PNG, BMP are supported
  • Check if a schedule is active and overriding manual playback — disable schedule to test
  • Check internal storage — A200 has 8GB; if full, delete old programs first
  • Check that the wall's receiving cards are getting data from the A200 output ports (indicator lights)

🌐 A200 Not Connecting to LEDMaster Cloud

Check in this order:
  • Confirm A200 has internet access — connect via WiFi Way 1 or Ethernet to router (Way 3)
  • In PlayerMaster Cloud settings — verify server URL is exactly correct (no typos, include https://)
  • Check account credentials — username and password are case-sensitive
  • Verify the terminal was created in the cloud portal before trying to link the device
  • Try switching from WiFi to wired Ethernet for a more stable cloud connection
  • Check if corporate firewall is blocking outbound traffic to colorlightcloud.com

🖥️ Screen Configuration — LEDSetting & iSet

📋

Two different jobs — two different tools

LEDSetting configures the receiving cards and LED hardware (what the wall shows). iSet configures the video processor (input source, output resolution, scaling). You typically need both for a full setup.

LEDSetting — Receiving Card Configuration

⚠️
Set PC NIC to static IP: 192.168.0.100, subnet 255.255.255.0. Connect PC to sending card via Ethernet. Disable other network adapters to avoid conflicts. Run LEDSetting as Administrator.
  1. Open LEDSetting → click Hardware Manager → confirm sending card is detected in the device list
  2. Click Receiving Card tab → select your receiving card model (i5A, i5A-F, K8, K10, E320, 5A-75E, etc.)
  3. Click Load from File → select the .RCVBP configuration file for your exact cabinet model. If you don't have the file, ask your cabinet manufacturer or use Intelligent Setting
  4. If no .RCVBP file: click Intelligent Setting → enter cabinet pixel width × height → select scan mode (check cabinet PCB or spec sheet) → select driver IC type (e.g., FM6363, ICN2037, MBI5124)
  5. Click Send to Hardware → wall should display a solid color test pattern. If garbled, adjust scan mode and OE polarity
  6. Go to Screen Connection tab → define the physical layout: how many cabinets wide × tall, cable entry point, and routing direction (left-to-right, snake, etc.)
  7. Under Sender Config: set the input resolution to match your signal source. Map the display area to cover your full wall
  8. Adjust Brightness, refresh rate, and gray level as needed for your environment
  9. Click Save to Hardware (EEPROM/Flash) — critical step so settings survive power loss
  10. Test with a full-color gradient and white field to check uniformity and dead pixels

iSet — Video Processor Configuration

  1. Connect PC to processor via Ethernet → open iSet → processor appears in the device list
  2. Input settings: select source type (HDMI/DVI/SDI/DP) → confirm resolution and refresh rate match your source. If source is 4K, verify the processor supports 4K input
  3. Output settings: set output canvas size to match the total pixel dimensions of your LED wall (e.g., 3840×1080 for a wide wall)
  4. Scaling: choose fill mode — Stretch (fills wall), Letterbox (preserves aspect ratio), or 1:1 pixel mapping
  5. Set Brightness, contrast, and color temperature for your environment (outdoor vs. indoor)
  6. If using multiple outputs (video splicer): define which portion of the canvas each output port drives
  7. Configure presets: name and save each configuration (e.g., "Stage Left HDMI 1080p", "Lobby 4K") for fast recall during events
  8. Test with signal source live — confirm no cropping, correct aspect ratio, and no color shifts
  9. Save preset to device memory — presets survive power cycles on Z-series and VX-series processors

⚙️ Software Reference

LEDSetting

Primary tool for configuring receiving cards. Load .RCVBP files, set scan parameters, map screen layout, save configs to hardware. Replaced LEDVISION (discontinued).

Works with: All i-series, K-series, E-series, 5A-series cards

iSet

Video processor control. Set input sources, output resolution, brightness, scaling, and presets. Real-time monitoring during events.

Works with: Z-series, VX-series, U-series processors

PlayerMaster

Windows software for managing A-series cloud players. Handles media, programs, schedules, screen parameters. Supports both LAN and Cloud modes.

Works with: A100, A200, A500, A800, A4K, C3 Pro

ColorAdept

Professional calibration software. Corrects brightness uniformity, white balance, and color accuracy across the full LED wall using a calibration camera.

Requires: Colorlight calibration camera (CCM6000 or similar)

GrandShow

Content creation and playback for CS-series media servers. Multi-layer video, live inputs, effects. For complex event productions.

Works with: CS16K, CS20-8K Pro media servers

LEDUpgrade

Batch firmware upgrade tool for sending and receiving cards. Use this to update multiple cards across a large installation quickly.

Always: back up config before upgrading firmware

LED Assistant (Mobile)

iOS and Android app for basic control of Colorlight cloud players and screen brightness adjustment from your phone.

Download: App Store / Google Play — search "LED Assistant"

LEDMaster (Cloud Portal)

Browser-based cloud management platform. Remote content publishing, scheduling, device monitoring, firmware updates, and team management.

Access: ledmaster.colorlightcloud.com
